# タスク1：3×3志望動機の作成

## 作成ルール

### 重要：自分の強みの織り交ぜ方

各志望動機の1点目、2点目、3点目の中に、**自分の強みをさりげなく織り交ぜる**こと。

**良い例：**
> プロダクト目線で事業全体の成果にコミットできる点です。GMV向上やCVR改善など、ビジネス成果と技術実装を直結させて考えられる環境は、**私がこれまで0→1で1500万DLを達成してきた経験**とも重なり、エンジニアとして最もやりがいを感じる部分です。

**悪い例：**
> プロダクト目線で事業全体の成果にコミットできる点です。〇〇〇〇〇。
> 
> **【自分の強み3つのアピール】**
> 私の〇〇、〇〇、〇〇という強みを活かして...

→ 強みを最後にまとめて述べるのではなく、各ポイントの説明の中に自然に入れ込む

---

### 具体的な作成ルール

1. **企業を3つ褒める + 自分の強みを織り交ぜる**
   - 各ポイント（1点目、2点目、3点目）で、企業の良いところを具体的に褒める
   - その中に、自分の強み・経験・実績をさりげなく入れ込む
   - 「〇〇という点です。〇〇〇〇（企業の魅力を説明）〇〇〇〇（自分の経験と結びつける）〇〇〇〇。」という流れ
   
2. **自然な流れを作る**
   - 企業の魅力を語る → 自分の経験と重ねる → 共感を示す、という自然な流れ
   - 強みをアピールしすぎず、企業を褒めることを主軸にする
   
3. **口調**：カジュアルで親しみやすく、かつ誠実な口調で書く
   - 「です・ます」調を基本とする
   - 自然で熱意が伝わる表現を使う

4. **長さ**：各ポイント100〜150字程度（3点で合計300〜450字程度）

## 参考例

以下は、実際に使用している志望動機の例です。この形式を参考にしてください：

### 【参考例1：プロダクト志向×技術チャレンジ】

#### 1点目（企業を褒める + 自分の強み①を織り交ぜる）
プロダクト目線で事業全体の成果にコミットできる点です。GMV向上やCVR改善など、ビジネス成果と技術実装を直結させて考えられる環境は、**私がこれまで0→1で1500万DLを達成してきた経験**とも重なり、エンジニアとして最もやりがいを感じる部分です。釣り人と釣船の双方の課題を理解し、データに基づいて機能開発の優先順位をつけながら、売上に直結する開発ができることに強く惹かれました。

#### 2点目（企業を褒める + 自分の強み②を織り交ぜる）
最新技術へのチャレンジ精神がある点です。App Routerが正式リリースから半年で採用を決断したという話を聞いて、技術的な意思決定のスピード感と、エンジニアと議論しながら技術選定をする文化に共感しました。**React/TypeScriptを4年間使ってきた経験**を活かして、Next.js 15 + React 19への移行など、常に最適な技術スタックを追求する環境でさらに成長できると感じています。

#### 3点目（企業を褒める + 自分の強み③を織り交ぜる）
釣り業界のDX化という社会的意義がある点です。電話予約の手間や受電対応の負担など、アナログな業界特有の課題をテクノロジーで解決し、業界全体を前進させていく。**BigQuery・Tableauを用いたデータ分析でKPI改善をしてきた経験**を活かして、このような0→1で業界を変えていく挑戦に貢献したいと強く思いました。

---

### 【参考例2：グローバル挑戦（自分の価値観を入れ込む例）】

#### 3点目（企業を褒める + 自分の価値観を織り交ぜる）
世界に挑戦している企業だからです。**親族に、マイクロソフトがライバルと認めていた会社の経営者の方がいて、マイクロソフトとの戦いに敗れた要因として、国内のみで戦っていたことが主要因だと思っています。**同じ失敗はしたくないのと、もっと世界で挑戦する企業を増やしていき、これからの世代に明るい世界を見せていきたい。
