# Agent Skills について

## 背景：エージェントの課題

現在のエージェントは賢いが、実際の業務に必要な専門知識が不足している。

## Agent Skills とは

この課題を解決するため、「Agent Skills」という概念を導入。エージェントの構築から、スキルの構築へと移行した。

**スキルの定義**：構成可能な手続き型知識をパッケージ化した、整理されたファイルの集まり（つまり「フォルダ」）

## スキルのメリット

- コードが自己文書化され、変更可能であるため、従来のツールの問題（不明瞭な指示、モデルによる変更不可など）を解決
- コンテキストウィンドウを保護し、数百のスキルを組み合わせ可能にするため、段階的に開示される
- 誰でも作成・利用できる

## スキルのエコシステム

ローンチから5週間で数千のスキルが生まれ、急速に成長している。

### スキルの種類

- 基盤スキル
- サードパーティースキル
- 企業・チーム内スキル

### 複雑化

単純なマークダウンファイルから、ソフトウェア、実行可能ファイル、コードなどをパッケージ化した複雑なスキルへと進化している。

### MCPサーバーとの連携

MCPサーバーが外部世界との接続を提供し、スキルが専門知識を提供する形で補完し合っている。

### 非技術者による作成

財務、採用、会計、法務などの非技術者によってスキルが構築されており、汎用エージェントのアクセス性を高めている。

## 汎用エージェントのアーキテクチャ

| コンポーネント | 役割 |
|---|---|
| エージェントループ | モデルの内部コンテキストとトークンの管理 |
| ランタイム環境 | ファイルシステムとコードの読み書き機能 |
| MCPサーバー | 外部のツールやデータとの接続 |
| スキルのライブラリ | 特定のタスク時にのみコンテキストに引き出される数百〜数千のスキル |

## 将来の展望

- **スキルをソフトウェアとして扱う**：テスト、評価、より良いツール、出力品質の測定
- **バージョニング**：スキルの進化とエージェントの動作変化を明確に追跡
- **依存関係**：他のスキル、MCPサーバー、パッケージへの明示的な依存関係と参照を可能にする
- **共有と配布**：企業や組織内での知識共有、さらにはコミュニティ全体での共有を通じて、エージェントの能力を向上させる
- **継続的な学習**：Claude自身がスキルを作成し、学習を将来のバージョンに転送することで、エージェントの性能を継続的に向上させる

## コンピューティングとの比較

- モデル → プロセッサ
- エージェントランタイム → オペレーティングシステム

プロセッサがOSによって価値を高められたように、モデルもエージェントランタイムによって価値を高められる。
