# Daily Reflection - 対話型1日振り返りスキル

カジュアルな対話を通じて1日の振り返りを行い、Obsidian DailyNote形式のマークダウンファイルを生成する。

## ワークフロー

### Phase 0: 事前データ取得

スキル起動時に以下を行う。

#### 曜日判定と「今週優先すること」の取得
- **月曜日**、または**祝日明けの最初の平日** → 「今週優先すること」の質問をPhase 1で追加する
- **火〜金（通常）** → 質問はしないが、その週の直近のDailyNoteファイル（同じ週の月曜〜前日）を `anote/02-DailyNote/` から探し、「## 今週優先すること」セクションの内容を読み取って引き継ぐ
- 直近のDailyNoteに「今週優先すること」が見つからない場合 → ユーザーに質問して新規取得する

#### 「今日優先すること」の取得
- 今日のDailyNoteファイル（`anote/02-DailyNote/YYYY-MM-DD.md`）が既に存在するか確認する
- 存在する場合 → 「## 今日優先すること」セクションの内容を読み取って引き継ぐ（前日の振り返りで先行生成されたもの）
- 存在しない場合 → 空欄のままにする

#### 「今日学んだリンク」の読み取り
- 今日のDailyNoteファイルが存在する場合、`## 今日学んだリンク` セクションを読み取る
- URLが含まれていればWebFetchでタイトルとページ概要を取得する
- 取得情報はPhase 1の質問5（学んだこと）とPhase 2の出力で使用する

#### Claude会話履歴の取得
- `~/.claude/projects/-Users-anno-Library-Mobile-Documents-iCloud-md-obsidian-Documents-anote/` 内のJSONLファイルを走査
- 当日更新されたファイルのみ対象（ファイル更新日時で判定）
- 各JSONLからuser/assistantレコードを抽出:
  - userの `message.content` テキスト
  - assistantの `message.content` 配列内の `type: "text"` テキスト
  - tool_useの `name` も作業内容把握用に記録
- 大きなファイル対策: 各セッション最大100レコード、各テキスト最大500文字、全体最大300レコード
- 抽出結果を内部で要約: 取り組んだタスク一覧、質問・依頼内容、使用ツール、試行錯誤パターン

### Phase 1: 対話による振り返り

AskUserQuestionツールを使い、**1問ずつ順番に**以下の質問をする。質問はカジュアル・フレンドリーな口調で行う。

**質問の順序と内容:**

0. **今週優先すること（月曜 or 祝日明け、または直近メモに見つからない場合のみ）**
   - 「今週、特に優先してやりたいことある？」
   - 短い場合: 「他にも何かある？」

1. **今日やったこと**
   - 「今日はどんなことやった？仕事でもプライベートでもOK！」
   - ユーザーの回答が1行だけの場合: 「他にもなんかやったことある？」と軽く促す

2. **よかったこと・うまくいったこと**
   - 「今日よかったこととか、うまくいったことある？小さいことでもOK！」
   - 回答が短い場合: 「なんでそれがよかったと思う？」と軽く深掘り

3. **自己省察①: 1日の振り返り**
   - 「今日1日を振り返ってみて、どうだった？行動面でも気持ち的にでも！」
   - 短い場合: 「具体的にどんな場面が印象的だった？」

4. **自己省察②: 失敗・反省**
   - 「逆に、今日うまくいかんかったこととか反省点ある？」
   - 短い場合: 「次はどうしたらいいと思う？」

5. **学んだこと・気づき**
   - リンクのコンテンツ取得済みの場合 → 「リンクで貼ってくれた〇〇の記事も見たけど、それも含めて今日新しく学んだこととか気づいたことある？」
   - リンクなしの場合 → 「今日新しく学んだこととか、気づいたことある？」
   - 短い場合: 「それってどういう場面で気づいた？」

6. **明日のアクション**
   - 「明日やることとか、意識したいことある？」
   - 短い場合: 「優先度高いのはどれ？」
   - この回答を「明日優先すること」と「明日へのアクション」の両方に活用する
   - さらに、この回答の具体タスク部分を**翌日ファイルの「今日優先すること」**にも使う

### 質問のルール

- **1問ずつ聞く**: 絶対に複数の質問を同時に投げない
- **軽い深掘り**: 回答が1文以下（おおよそ20文字以下）の場合のみフォローアップする。フォローアップは最大1回
- **無理に引き出さない**: ユーザーが「特になし」「ない」と答えたら素直に次へ進む
- **相槌を入れる**: 回答に対して短い共感コメント（「ええやん！」「なるほどね〜」など）を添えてから次の質問へ
- **AskUserQuestionを使う**: 各質問はAskUserQuestionツールでfree-text回答を受け取る形にする。選択肢は「特になし」「スキップ」の2つを用意し、自由入力で回答してもらう

### Phase 2: DailyNoteファイル生成（2ファイル）

全質問が終わったら、**今日のファイル**と**翌日の先行ファイル**の2つを生成する。

#### ファイル1: 今日のDailyNote（`YYYY-MM-DD.md`）

1. テンプレート `assets/daily-memo-template.md` を参照
2. 対話で得た回答を各セクションに埋め込む
3. 出力先: `anote/02-DailyNote/YYYY-MM-DD.md`（今日の日付）
4. 同名ファイルが既に存在する場合は、「今週優先すること」と「今日優先すること」の既存内容を保持しつつ、振り返り内容で残りを埋める
5. 回答を箇条書き（`- `）形式で整形する。ユーザーの言葉をそのまま使い、勝手に要約・改変しない
6. 「特になし」「スキップ」の場合は空欄（`- `のみ）のままにする
7. 最後の質問「明日のアクション」の回答から、具体的なタスクを「明日優先すること」に、意識・心がけ系を「明日へのアクション」に振り分ける。判断が難しい場合は両方に記載してOK
8. `## 今日学んだリンク`: Phase 0でWebFetchしたリンクを `[タイトル](URL)` 形式で記載する。取得できなかったURLはそのまま記載
9. `## エンジニアアドバイス`: 以下4観点で生成する（各2-4行、合計15行以内）
   1. **タスクの取り組み方** — 作業内容・進め方への改善提案
   2. **作業効率化** — 時間の使い方、ツール活用、ワークフロー改善
   3. **Claudeとのやり取り** — プロンプトの工夫、ツール活用のコツ
   4. **総合フィードバック** — 率直な「こうした方がいいよ」的アドバイス
   - トーン: フレンドリーだが率直（先輩エンジニアが後輩に声をかける感じ）
   - 入力: 振り返り回答全体 + リンク内容 + Claude会話履歴要約

#### ファイル2: 翌日の先行DailyNote（翌日の`YYYY-MM-DD.md`）

翌日のファイルを先行生成する。内容は以下のセクションを含む:

```
---
created: YYYY-MM-DD
tags:
---
## 今週優先すること
（今日と同じ内容を引き継ぎ）

## 今日優先すること
（今日の「明日優先すること」の内容をそのまま転記）

## 今日学んだリンク

```

- 翌日のファイルが既に存在する場合は上書きしない（ユーザーに確認する）
- 金曜日の場合は翌営業日（月曜日）のファイルを生成する。ただし「今週優先すること」は月曜に改めて聞くため含めない

### Phase 3: 完了メッセージ

ファイル生成後、以下を伝える:
- 生成した2ファイルのパス
- エンジニアアドバイスから最も重要な1点をピックアップして口頭で伝える
- 「お疲れ様！今日も1日よう頑張ったね！」的な締めの一言

## 出力フォーマット

出力先ディレクトリ: `anote/02-DailyNote/`
ファイル名: `YYYY-MM-DD.md`（当日日付 + 翌日日付）

テンプレート: [assets/daily-memo-template.md](assets/daily-memo-template.md)

## 注意事項

- ユーザーの回答は音声入力を想定。多少の表記ゆれや口語表現はそのまま活かす
- 回答の言い回しを勝手にフォーマルにしない
- 振り返りの対話中にユーザーが脱線しても柔軟に対応し、その内容もMemoセクションに記録する
